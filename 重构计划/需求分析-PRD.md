# Zoey v2 PRD（产品需求文档）

## 1. 项目定位

一句话：**基于阿里云 DashScope API 的多模态 AI 对话助手，支持桌面和移动端浏览器使用，可部署到 Hugging Face Spaces。**

## 2. 用户画像

**只有一类用户：普通用户。**
不需要知道模型是什么、不需要切换模型。打开网页 → 打字/发图/发语音 → 得到回复。模型固定，用户零感知。多人在不同设备上同时使用，彼此数据隔离。

## 3. 功能清单

### P0（必须）

| 功能 | 说明 |
|------|------|
| 文字对话 | 输入文字，AI 流式回复 |
| 图片输入 | 上传图片让 AI 理解 |
| 音频输入 | 上传音频让 AI 理解 |
| 流式输出 | AI 回复逐字显示 |
| 多轮对话 | 保持上下文连贯 |
| 多会话管理 | 左侧边栏显示对话列表，支持新建、切换、删除会话 |
| 自动生成标题 | 每个会话根据首条消息自动生成标题 |
| 历史记录持久化 | 存储在浏览器 localStorage，刷新页面不丢失 |
| 删除确认 | 删除会话前弹出确认框 |
| 移动端响应式布局 | 手机浏览器可正常使用 |

### P1（重要）

| 功能 | 说明 |
|------|------|
| 系统提示词 | 自定义 AI 角色/行为设定 |
| 预设问题 | 一键提问常用问题 |
| 状态反馈 | 显示"正在思考"、"就绪"等状态 |
| 错误恢复 | API 失败时给出明确提示，不崩溃 |

## 4. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 首 token 延迟 < 2 秒（流式） |
| 兼容性 | Chrome / Edge / Firefox / 手机浏览器最新版 |
| 可维护性 | 模块分层清晰，单文件不超过 300 行 |
| 安全性 | API 密钥仅存于服务端环境变量，不暴露给客户端 |
| 可测试性 | 核心逻辑单元测试覆盖率 ≥ 80% |

## 5. 系统约束

- **固定模型**：使用 `qwen3.5-omni-plus-2026-03-15`，可处理文字/图片/音频输入。不支持多图同传、多音频同传、图+音混合——遇到时直接提示用户
- **网络约束**：依赖公网 API，离线不可用
- **平台部署**：主推 Hugging Face Spaces，免费版 48 小时无访问自动休眠（约 30 秒唤醒），可接受
- **文件大小**：图片限制 10MB，音频限制 25MB，超出时提示用户压缩后重传

## 6. 数据存储方案

| 数据 | 存在哪 | 说明 |
|------|--------|------|
| 多会话列表 | 浏览器 localStorage | 每个会话含消息数组、标题、时间戳 |
| 当前会话 ID | 浏览器 localStorage | 记录用户当前打开的是哪个会话 |
| API 密钥 | 服务端环境变量 | 部署时配置，不暴露给浏览器 |
| 用户设置 | 浏览器 localStorage | 系统提示词等偏好设置 |

多人同时使用同一 URL 时，各自的会话列表互不干扰（存在各自浏览器里）。

## 7. 非功能范围（明确不做）

- ❌ 单条消息删除
- ❌ 清除全部对话
- ❌ 语音回复
- ❌ 模型选择/切换 UI
- ❌ 跨设备历史同步
- ❌ 用户登录/认证系统
- ❌ 服务端数据库
- ❌ 国际化/多语言（仅中文）
- ❌ 插件/扩展系统
- ❌ 对话导出/分析

## 8. 典型用户流程

### 单人桌面使用

```
打开 http://localhost:7860
         │
         ▼
页面加载 → 读取 localStorage → 显示上次的会话列表和当前对话
         │
         ▼
在输入框打字 "介绍一下你自己" → AI 流式回复
         │
         ▼
会话自动生成标题 "介绍一下你自己"
         │
         ▼
点击 [+ 新对话] → 切换到空白对话 → 上传图片 → AI 分析
         │
         ▼
左侧列表出现两个对话，可点击切换
         │
         ▼
鼠标悬停某对话 → 点击 ✕ → 确认弹窗 → 删除该对话
         │
         ▼
关闭浏览器，第二天打开 → 两个对话都在，可继续聊
```

### 多人同时使用

```
用户A（电脑） → 访问 URL → 自己的会话列表
用户B（手机） → 访问同一 URL → 自己建自己的会话，不影响 A
用户C（电脑） → 访问同一 URL → 同上
```

### 部署到 Hugging Face Spaces

```
开发者: git push → Spaces 自动同步
用户:   访问 https://huggingface.co/spaces/xxx/zoey → 直接使用
开发者: 不用操心服务器运维，免费额度够用
```
