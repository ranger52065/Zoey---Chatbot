### 模块依赖关系

```
app.py
  ├── core/message.py  (会话管理、消息管理)
  │     └── core/processor.py  (文件处理)
  └── api/client.py  (API 通信)
```

严格单向依赖：**app → core → api**，不存在反向引用。

## 2. 数据模型

### 2.1 会话结构

```python
# 一个完整的对话
{
    "id": "conv_001",                              # 唯一 ID
    "title": "介绍一下你自己",                       # AI 自动生成的标题
    "messages": [                                   # 消息列表
        {"role": "user", "content": "介绍一下你自己"},
        {"role": "assistant", "content": "你好！我是Zoey..."}
    ],
    "created_at": "2026-06-15T10:30:00",           # 创建时间
    "updated_at": "2026-06-15T10:35:00",           # 最后更新时间
}
```

### 2.2 localStorage 存储结构

```javascript
// 键名: "zoey_conversations"
// 值: JSON 序列化的会话列表（按 updated_at 倒序）
[
    { "id": "conv_003", "title": "帮我看看这张图", "messages": [...], ... },
    { "id": "conv_002", "title": "讲个笑话", "messages": [...], ... },
    { "id": "conv_001", "title": "介绍一下你自己", "messages": [...], ... }
]

// 键名: "zoey_current_conv_id"
// 值: 当前打开的会话 ID
"conv_003"

// 键名: "zoey_system_prompt"
// 值: 字符串
"你是一个友好的AI助手"
```

### 2.3 自动生成标题策略

| 场景 | 标题来源 |
|------|---------|
| 用户发送第一条文字消息 | 取前 20 个字符作为标题 |
| 用户上传图片 | 取"[图片] 文件名"作为标题 |
| 用户上传音频 | 取"[音频] 文件名"作为标题 |
| AI 回复后 | 由 AI 生成摘要标题（后续优化，第一期用上述规则） |

### 2.4 文件上传处理

文件仅存在于运行时，**不序列化到 localStorage**：

```python
# 用户上传文件 → Gradio 临时路径 → 处理后转为 API 格式
{
    "type": "image_url",
    "image_url": {"url": "data:image/jpeg;base64,...", "detail": "low"}
}

{
    "type": "input_audio",
    "input_audio": {"data": "base64...", "format": "wav"}
}
```

## 5. 事件绑定

```
用户操作                   触发函数              数据流
─────────               ────────              ─────
页面加载                 load_conversations()    从 localStorage 读取
                          load_current_conv()
                          显示到界面

点击 [+ 新对话]         create_conversation()    创建新会话对象
                          set_current_conv()      切换到新会话
                          save_conversations()    保存
                          清空聊天窗口

点击对话列表某条         set_current_conv()      切换当前会话 ID
                          load_conversations()    读取该会话消息
                          显示到聊天窗口

悬停对话列表 ✕          confirm_dialog()        显示确认弹窗
  → 确认删除             delete_conversation()   删除该会话
                          save_conversations()    保存
                          刷新列表（如果删除的是当前会话则清空聊天窗口）

提交消息 (chat_input)    on_user_message()       追加到当前会话 messages
                          │
                          ▼
                        on_submit()              build_api_messages()
                          │                        → processor 处理文件
                          ▼
                        validate_input_types()   不合法 → 返回提示
                          │
                          ▼
                        stream_chat()            流式 yield 更新界面
                          │
                          ▼
                        update_conversation_title()  首次回复后生成标题
                        save_conversations()     保存到 localStorage

设置变更 (系统提示词)    save_conversations()     保存到 localStorage
```

## 6. 文件大小验证流程

```
用户上传文件
     │
     ▼
Gradio 过滤 file_types → 仅允许 image / .wav / .mp3 / .m4a
     │
     ▼
processor.py 检查文件大小
     │
     ├── 图片 > 10MB → 打印警告 → 该文件跳过，前端显示"文件过大"
     ├── 音频 > 25MB → 打印警告 → 该文件跳过，前端显示"文件过大"
     └── 正常 → 正常处理
```

## 7. 错误处理策略

| 错误类型 | 表现 |
|---------|------|
| API 密钥无效 | 聊天窗口显示 "❌ API 密钥无效，请检查配置" |
| 网络超时 | 显示 "❌ 请求超时，请检查网络连接" |
| API 限流 | 显示 "❌ 请求过于频繁，请稍后重试" |
| 文件处理失败 | 显示 "❌ 文件处理失败，请检查文件是否损坏" |
| 未知错误 | 显示 "❌ 出错了: {错误信息}" + 打印详细日志 |

## 8. 编码顺序

```
第一阶段（API + 文件处理 — 无依赖）
  api/client.py       ← DashScope 通信
  core/processor.py   ← 图片/音频处理

第二阶段（会话管理 — 依赖 processor）
  core/message.py     ← 会话 CRUD、消息构建、约束校验

第三阶段（UI 和事件绑定 — 依赖 message 和 api）
  app.py              ← 侧边栏 + 聊天区 + 全部事件

第四阶段（测试）
  tests/              ← 单元测试覆盖核心逻辑
```
